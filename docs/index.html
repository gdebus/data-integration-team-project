Introduction

Welcome to the documentation for our team project. This site explains how we use LLM-based agents to build robust and scalable data integration pipelines.

Status

This project is currently under active development. Some features may still be experimental.

Project Overview

Data integration is the process of combining data from multiple sources into a unified view. Our project explores how modern Large Language Models (LLMs) can be integrated into traditional pipelines to handle complex tasks like schema matching, entity resolution, and data fusion.

Core Goals

  • Automation: Reduce manual intervention in mapping and merging data.
  • Accuracy: Leverage LLM reasoning to improve matching precision.
  • Scalability: Design a pipeline that handles diverse datasets from multiple domains.

Architecture

Our pipeline follows a modular design, separating schema matching, entity matching, and data fusion.

Workflow Overview

  1. Data Loading: Ingesting datasets in various formats (Parquet, CSV).
  2. Schema Matching: Mapping attributes from different sources to a target schema.
  3. Entity Matching: Identifying records that represent the same real-world entity.
  4. Data Fusion: Merging matching records into a single, high-quality representation.

Data Fusion

Data Fusion is the final stage of our integration pipeline, where we reconcile conflicting values and merge information.

Strategy

  • Conflict Resolution: We use LLM-based reasoning to choose the most accurate value when sources disagree.
  • Normalization: Standardizing formats for dates, currencies, and names.
